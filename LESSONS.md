# 项目经验教训 — 下次开发前必读

> 从 Barley Grid · F1 Data Center 项目中总结的实战规则

---

## ✅ 做得好的（继续保持）

### 视觉设计
- **从克制开始**：色板限制在 4 色以内（底色、文字色、2 个点缀色），不够再加
- **用一个基类管所有卡片**：`.apple-card` 统一了圆角、阴影、边框，避免了组件间风格不一致
- **系统字体优先**：`-apple-system, Inter, Segoe UI` 免加载、渲染快、跨端一致

### 性能
- **动画分层**：毛玻璃（blur）和运动（transform）放在不同的 DOM 层，互不干扰，确保 60fps
- **GPU 加速标记**：需要动画的元素加 `transform-gpu` + `will-change-transform`
- **先确认性能可行，再决定视觉方案**：不做"好看但卡"的东西

### 架构
- **数据顶层拉取，props 层层传递**：避免重复请求、数据不一致、加载状态分散
- **能用原生就不引库**：没有用 Redux / Router / UI 库，项目规模不需要
- **Drawer 而非路由跳转**：符合"预览后返回"的用户心智，体验更流畅

### 流程
- **MVP 验证完立即切真实 API**：越早用真实数据，越早暴露边界情况
- **小步提交**：每次改动清晰、可控、可回滚

## ❌ 踩过的坑（下次避免）

### 1. 不要用 `overflow: hidden` 锁定滚动
- **问题**：移动端浏览器会在设置 `overflow: hidden` 时丢失 `scrollY`
- **正确做法**：用 `position: fixed + top 偏移` 方案（已封装在 `scrollLock.js`）
- **规则**：**所有需要锁定滚动的场景（Drawer、Modal、全屏弹窗）都用 `lockScroll()` / `unlockScroll()`**

### 2. 不要对特定排名硬编码颜色
- **问题**：第一名强制红色，当第一名不是法拉利时语义错误
- **正确做法**：颜色始终从数据（`teamColor`）中取，想突出排名用粗细/大小/动画，不改颜色
- **规则**：**有语义的视觉属性（颜色 = 车队归属）不要被无关逻辑覆盖**

### 3. 不要用 `setTimeout` 做跨组件时序同步
- **问题**：`setTimeout(150ms)` 恢复滚动位置，在不同设备上时机不稳定
- **正确做法**：把相关操作封装到同一个函数中（如 `unlockScroll` 内部同步恢复），避免竞态
- **规则**：**如果两个操作有因果关系，放在一起执行，不要用延时猜**

### 4. 视觉迭代不要推翻重来
- **问题**：前 4 轮视觉迭代每次都大规模重写，浪费了大量时间
- **正确做法**：先定一个极简基础版，然后只做增量调整
- **规则**：**先出灰度线框 → 确认布局和信息层级 → 最后上色上材质**

### 5. `backdrop-filter: blur()` 不要放在运动元素上
- **问题**：GPU 每帧重算模糊区域，导致动画掉帧
- **正确做法**：blur 放在固定定位的独立层，只做 opacity 切换
- **规则**：**blur 不动，动的不 blur**

## 📋 新项目启动 Checklist

开始新项目前，过一遍这个清单：

- [ ] 色板定了吗？不超过 4 色
- [ ] 卡片/容器有统一基类吗？
- [ ] 用系统字体还是自定义字体？确认了吗？
- [ ] 数据在哪一层拉取？是否统一管理？
- [ ] 需要弹窗/抽屉吗？滚动锁定用 `position: fixed` 方案
- [ ] 有动画需求吗？blur 和 transform 是否分层了？
- [ ] 颜色有语义含义吗？确保不被硬编码覆盖
- [ ] 移动端测试了吗？iOS Safari 单独验证过吗？
